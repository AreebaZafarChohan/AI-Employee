# Daily Briefing Generator

You are executing the **Daily Briefing Generator** skill. Follow every step precisely.

## Step 1 — Read Data Sources

### 1.1 Business Goals
Read `AI-Employee-Vault/Company_Handbook.md`:
- Extract sections with "Goal", "Objective", or "Mission" in headers
- List all bullet points under these sections
- Maximum 5 goals for summary

### 1.2 Completed Tasks
Read `AI-Employee-Vault/Done/*.md` files from last 24 hours:
- Check `completed_at`, `executed_at`, `created_at` fields in frontmatter
- Fall back to file modification time
- Extract: title, type, priority, body preview (first 500 chars)

### 1.3 Accounting Data
Read `AI-Employee-Vault/Accounting/*.md` (if exists) and vault root:
- Look for files with "account", "invoice", "payment" in name
- Extract transaction metadata (amount, recipient, date)
- Find revenue signals ($, revenue, paid, invoice keywords)

## Step 2 — Generate Briefing Sections

### 2.1 Executive Summary
```markdown
## Executive Summary
- Generated timestamp
- List active business goals (up to 5)
- Task completion count
- Breakdown by task type
```

### 2.2 Revenue Snapshot
```markdown
## Revenue Snapshot
- Recent transactions (up to 5) with amounts
- Revenue signals from document snippets
- Or "No recent transactions" if none
```

### 2.3 Task Summary
```markdown
## Task Summary
For each task (up to 10, most recent first):
- Formatted title (replace - with spaces, title case)
- Completion date
- Source file reference
- Content preview (first 5 lines)
```

### 2.4 Bottlenecks & Blockers
```markdown
## Bottlenecks & Blockers
Identify:
- High priority items count
- Pending approval items count
- Goal progress (tasks vs goals ratio)
- Any misalignments
```

### 2.5 Suggestions & Next Steps
```markdown
## Suggestions & Next Steps
Always include:
1. Review pending items in Needs_Action/
2. Align with active business goals
3. Follow up on emails if any processed
4. Weekly review if >= 5 tasks completed
```

## Step 3 — Save Briefing

Create file: `AI-Employee-Vault/Briefings/YYYY-MM-DD_Daily.md`

Frontmatter:
```yaml
---
briefing_id: "YYYYMMDD"
generated_at: "ISO-timestamp"
type: daily_briefing
period: 24h
---
```

Body: All 5 sections from Step 2.

Footer:
```markdown
---
*Generated by AI Employee Daily Briefing Generator*
```

## Step 4 — Update Dashboard

Read `AI-Employee-Vault/Dashboard.md`.

If "## Daily Briefings" section exists:
- Add new briefing link: `- [[YYYY-MM-DD_Daily.md]]`

If section doesn't exist:
- Create before "## Quick Links":
```markdown
## Daily Briefings

- [[YYYY-MM-DD_Daily.md]]
```

Save updated Dashboard.md.

## Step 5 — Log Generation

Append to `AI-Employee-Vault/Logs/daily-briefing-YYYY-MM-DD.log`:

```json
{
  "timestamp": "ISO-timestamp",
  "briefing_id": "YYYYMMDD",
  "status": "success",
  "dry_run": false,
  "briefing_file": "path/to/briefing",
  "goals_read": count,
  "tasks_read": count
}
```

## Error Handling

### Missing Company_Handbook
- Continue with empty goals list
- Log warning

### No Recent Tasks
- Show "No tasks completed in last 24 hours"
- Continue generation

### Dashboard Locked
- Log error
- Continue (briefing already saved)
