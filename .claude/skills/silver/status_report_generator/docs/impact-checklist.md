# Status Report Generator - Impact Checklist

## Overview
This checklist helps assess the quality, accuracy, and overall impact of status reports generated by the `status_report_generator` skill. It covers aspects from data integrity to stakeholder confidence.

---

## 1. Data Integrity and Accuracy

### Pre-Generation Checks

-   [ ] **Data Source Reliability:** Are all data sources (project management tools, approval systems, metric APIs) trusted and authoritative?
-   [ ] **Data Freshness:** Is the data retrieved within the specified reporting period and before the stale data threshold?
-   [ ] **Data Completeness:** Are all expected data points for tasks, approvals, and milestones present, or are missing values handled transparently?
-   [ ] **Data Aggregation Logic:** Is the logic for aggregating data (e.g., counting open tasks, summarizing milestone progress) correct and verified?
-   [ ] **Consistency of Definitions:** Are terms like "task status," "priority," and "milestone completion" consistently defined and applied across all data sources?

### Post-Generation Validation

-   [ ] **Numerical Accuracy:** Do all counts, percentages, and numerical values in the report accurately reflect the source data? (Spot check critical metrics/counts)
-   [ ] **Categorical Correctness:** Are statuses (e.g., RAG, task status, approval status) correctly mapped and represented?
-   [ ] **Timestamps:** Are all timestamps (report date, data last updated) correct and in the expected format/timezone?
-   [ ] **Missing Data Handling:** If data was missing, was it clearly indicated (e.g., "N/A", "Not Available") rather than omitted silently or filled with incorrect assumptions?
-   [ ] **Stale Data Warning:** If `STATUS_REPORT_WARN_ON_STALE_DATA` is enabled, does the warning appear correctly when data exceeds the threshold?

---

## 2. Clarity and Comprehension

### Layout and Structure

-   [ ] **Logical Flow:** Does the report follow a logical progression, starting with an executive summary and moving to details?
-   [ ] **Section Headings:** Are headings clear, concise, and reflective of the content? Is the numbering consistent?
-   [ ] **Readability:** Is the overall layout clean, with appropriate use of whitespace, bullet points, and tables for easy scanning?
-   [ ] **Table of Contents:** Is the auto-generated TOC accurate and helpful for navigation?

### Content Readability

-   [ ] **Conciseness:** Is the information presented as concisely as possible, summarizing key points rather than providing raw data?
-   [ ] **Terminology:** Is jargon minimized, or are technical terms explained for the target audience?
-   [ ] **Key Takeaways:** Are the most important messages, highlights, and lowlights easily identifiable?
-   [ ] **Actionability:** Does the report clearly highlight required actions, decisions, risks, or dependencies for the reader?

---

## 3. Communication Effectiveness

### Stakeholder Alignment

-   [ ] **Audience Relevance:** Is the content tailored to the needs and expectations of the primary report audience (e.g., executive vs. team lead)?
-   [ ] **Transparency:** Does the report provide an honest and balanced view of project status, including challenges and risks?
-   [ ] **Confidence Building:** Does the report instill confidence in project management and execution?

### Decision Support

-   [ ] **Informed Decisions:** Does the report provide sufficient and accurate information for stakeholders to make timely and informed decisions?
-   [ ] **Proactive Problem Solving:** Does it highlight potential issues, risks, or dependencies early enough for proactive intervention?

---

## 4. Operational Impact

### Efficiency

-   [ ] **Automation Value:** Does the automated generation significantly reduce the manual effort and time spent by project managers/leads on report compilation?
-   [ ] **Meeting Effectiveness:** Does the report streamline discussions in status meetings, allowing more focus on problem-solving rather than data gathering?

### Compliance and Audit

-   [ ] **Audit Trail:** Does the report include clear information on when and by whom it was generated, and from which data sources, for auditability?
-   [ ] **Record Keeping:** Does it serve as a reliable historical record of project progress and decisions?

---

## 5. Maintenance and Scalability

-   [ ] **Template Robustness:** Can the report template handle variations in data structure or project complexity without breaking?
-   [ ] **Data Source Flexibility:** Is it straightforward to integrate new data sources or update existing ones?
-   [ ] **Performance:** Does the generation process complete within acceptable timeframes, even for large reports or frequent generation?
-   [ ] **Ease of Update:** How easy is it to update the template or underlying data fetching logic as project needs evolve?
