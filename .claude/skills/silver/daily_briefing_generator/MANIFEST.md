# Daily Briefing Generator — Manifest

**Created:** 2026-02-25
**Domain:** silver
**Status:** Production Ready
**Version:** 1.0.0

## Overview

The `daily_briefing_generator` skill automatically generates comprehensive daily briefings from vault data. It reads business goals, completed tasks, and accounting data, then produces an executive summary with actionable insights.

## Integration Points

### Upstream
- **Company_Handbook.md** — Business goals and objectives
- **/Done/*.md** — Completed tasks (last 24h)
- **/Accounting/*.md** — Financial transactions
- **Dashboard.md** — Updated with briefing links

### Downstream
- **/Briefings/** — Generated briefing files
- **/Logs/** — Generation audit trail
- **Human stakeholders** — Daily status awareness

### Data Flow
```
Company_Handbook ──┐
/Done ─────────────┼──→ daily_briefing_generator ──→ /Briefings/
/Accounting ───────┘                                    ↓
                                                   Dashboard.md
```

## Configuration

Add to `.env`:
```bash
# Daily Briefing Generator
BRIEFING_LOOKBACK_HOURS=24
BRIEFING_DRY_RUN=false
```


- **Format:** JSON lines
- **Frequency:** Per generation

## Troubleshooting

### No briefing generated
1. Check Python is installed: `python --version`
2. Verify PyYAML: `pip show pyyaml`
3. Check vault path exists
4. Run with DEBUG logging: `LOG_LEVEL=DEBUG python daily_briefing_generator.py`

### Dashboard not updated
1. Verify Dashboard.md exists
2. Check file permissions
3. Ensure no other process has file locked

### Missing data in briefing
1. Verify source files exist (Company_Handbook, /Done)
2. Check file timestamps (within lookback period)
3. Review frontmatter format in source files
