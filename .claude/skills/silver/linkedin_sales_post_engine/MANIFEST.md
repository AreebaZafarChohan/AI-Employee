# LinkedIn Sales Post Engine — Manifest

**Created:** 2026-02-25
**Domain:** silver
**Status:** Production Ready
**Version:** 1.0.0

## Overview

The `linkedin_sales_post_engine` skill automatically generates LinkedIn sales post drafts from vault data. It reads business goals, revenue summaries, and completed tasks, then produces engaging posts (≤180 words) with CTAs for human review.

## Integration Points

### Upstream
- **Company_Handbook.md** — Business goals and objectives
- **Dashboard.md** — Revenue metrics and highlights
- **/Done/*.md** — Completed tasks (last 5)
- **/Accounting/*.md** — Financial transactions

### Downstream
- **/Pending_Approval/** — Draft posts awaiting review
- **/Approved/** — Approved posts ready for publishing
- **/Rejected/** — Rejected posts with feedback
- **/Logs/** — Generation audit trail

### Data Flow
```
Company_Handbook ──┐
Dashboard ─────────┼──→ linkedin_sales_post_engine ──→ /Pending_Approval/
/Done ─────────────┘                                       │
                                                           │
                                                    Human Review
                                                           │
                                              ┌────────────┼────────────┐
                                              │                         │
                                              ↓                         ↓
                                         /Approved                 /Rejected
                                              │
                                         (Publish manually)
```

## Troubleshooting

### No post generated
1. Check Python is installed: `python --version`
2. Verify PyYAML: `pip show pyyaml`
3. Check vault path exists
4. Run with DEBUG logging: `LOG_LEVEL=DEBUG python ...`

### Post too generic
1. Add business goals to Company_Handbook.md
2. Complete more tasks (appear in /Done)
3. Update Dashboard with revenue metrics

### Word count exceeds limit
- Automatic truncation applied
- Check logs for truncation warnings
- May need to reduce task descriptions

## Security

- **No auto-publishing** — Always requires human approval
- **Draft only** — Saved to Pending_Approval/
- **Audit trail** — All generations logged
- **DRY_RUN support** — Test without file writes
