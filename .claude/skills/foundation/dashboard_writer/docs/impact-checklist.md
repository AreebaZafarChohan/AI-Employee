# Dashboard Writer - Impact Checklist

## Overview
This checklist helps assess the quality, accuracy, and overall impact of dashboards generated by the `dashboard_writer` skill. It covers various aspects from data integrity to audience comprehension.

---

## 1. Data Integrity and Accuracy

### Pre-Generation Checks

-   [ ] **Data Source Reliability:** Is the source data (JSON, YAML, API) from a trusted and authoritative system?
-   [ ] **Data Freshness:** Is the data up-to-date and within acceptable latency limits for the reporting period?
-   [ ] **Data Completeness:** Are all expected data points present, or are missing values handled gracefully?
-   [ ] **Data Transformation:** Are any data transformations (e.g., aggregation, filtering) correctly applied and validated?
-   [ ] **Thresholds & Definitions:** Are all metric thresholds and definitions clearly stated and consistently applied?

### Post-Generation Validation

-   [ ] **Numerical Accuracy:** Do all numerical values in the dashboard match the source data? (Spot check critical metrics)
-   [ ] **Categorical Correctness:** Are categories (e.g., task statuses, approval types) correctly represented?
-   [ ] **Trend Accuracy:** Are reported trends (e.g., increasing, decreasing, stable) reflective of the underlying data?
-   [ ] **Missing Data Handling:** If data was missing, was it clearly indicated (e.g., "N/A", "Unknown") rather than omitted silently?

---

## 2. Clarity and Comprehension

### Layout and Structure

-   [ ] **Logical Flow:** Does the dashboard follow a logical narrative from executive summary to detailed sections?
-   [ ] **Section Headings:** Are headings clear, concise, and reflective of the content?
-   [ ] **White Space:** Is there sufficient white space to prevent information overload?
-   [ ] **Consistency:** Is formatting (e.g., table styles, bullet points) consistent throughout?

### Content Readability

-   [ ] **Conciseness:** Is information presented as concisely as possible without losing meaning?
-   [ ] **Terminology:** Is jargon avoided, or are technical terms clearly explained for the target audience?
-   [ ] **Key Takeaways:** Are the most important messages and insights easily identifiable?
-   [ ] **Actionability:** Does the dashboard clearly indicate what actions, if any, are required from the reader?

---

## 3. Visual Impact and Effectiveness

### Tables and Lists

-   [ ] **Column Headers:** Are table headers descriptive and unambiguous?
-   [ ] **Data Alignment:** Are numerical data right-aligned, and text left-aligned in tables for readability?
-   [ ] **List Clarity:** Are bullet points and numbered lists easy to follow?

### Diagrams (Mermaid)

-   [ ] **Relevance:** Does the diagram add value and clarify complex information?
-   [ ] **Simplicity:** Is the diagram easy to understand at a glance?
-   [ ] **Accuracy:** Does the diagram accurately represent the data or process?
-   [ ] **Accessibility:** Is the diagram understandable even without color (for accessibility)?

---

## 4. Operational and Business Impact

### Decision Support

-   [ ] **Informed Decisions:** Does the dashboard provide sufficient information for stakeholders to make informed decisions?
-   [ ] **Early Warning:** Does it highlight potential issues or risks proactively?
-   [ ] **Opportunity Identification:** Does it reveal areas for improvement or new opportunities?

### Efficiency

-   [ ] **Time Saving:** Does the automated generation save significant manual effort compared to previous methods?
-   [ ] **Meeting Effectiveness:** Does the dashboard streamline discussions in meetings by presenting facts clearly?

### Compliance and Audit

-   [ ] **Audit Trail:** Is there clear information on when and by whom the dashboard was generated, and from which data sources?
-   [ ] **Data Governance:** Does the dashboard adhere to data privacy and access control policies?

---

## 5. Maintenance and Scalability

-   [ ] **Template Robustness:** Can the template handle variations in data structure or volume without breaking?
-   [ ] **Data Source Flexibility:** Is it easy to switch or add new data sources?
-   [ ] **Performance:** Does the generation process complete within acceptable timeframes, especially for large datasets?
-   [ ] **Ease of Update:** How easy is it to update the template or underlying data fetching logic?
