## Core Entities

### Task

A unit of work to be managed by the AI Employee system.

```typescript
// src/types/task.ts

export type TaskStatus = 'pending' | 'in-progress' | 'completed' | 'cancelled';

export interface Task {
  id: string;              // UUID v4
  title: string;           // 1-200 characters
  description: string;     // 0-2000 characters
  status: TaskStatus;
  createdAt: string;       // ISO-8601 datetime
  updatedAt: string;       // ISO-8601 datetime
}

export interface CreateTaskInput {
  title: string;
  description?: string;    // Optional
}

export interface UpdateTaskInput {
  status?: TaskStatus;
  title?: string;
  description?: string;
}
```

**Validation Rules**:
- `title`: Required, 1-200 characters
- `description`: Optional, 0-2000 characters
- `status`: Must be one of the four defined enum values

**State Transitions**:
```
pending → in-progress → completed
                      → cancelled
pending → cancelled
```

### Plan

A structured output generated by the AI that provides guidance on approaching tasks.

```typescript
// src/types/plan.ts

export interface Plan {
  id: string;            // UUID v4
  taskId: string;        // UUID v4 (references Task)
  sections: PlanSection[];
  summary: string;       // 1-500 characters
  createdAt: string;     // ISO-8601 datetime
}

export interface PlanSection {
  title: string;         // 1-100 characters
  steps: string[];       // 1-20 steps, each 1-500 characters
}

export interface GeneratePlanInput {
  taskIds: string[];     // Array of task UUIDs to plan for
}

export interface PlanGenerationStatus {
  status: 'processing';
  estimatedCompletion?: string;  // ISO-8601 datetime (optional)
}
```

**Validation Rules**:
- `summary`: Required, 1-500 characters
- `sections`: Required, 1-10 sections
- `steps` per section: 1-20 steps

### System State

The current operational state of the AI agent.

```typescript
// src/types/system-state.ts

export type AgentStatus = 'idle' | 'processing' | 'working' | 'error';

export interface SystemState {
  status: AgentStatus;
  lastActivityAt: string;    // ISO-8601 datetime
  currentTaskId: string | null;  // UUID v4 or null if idle
}
```

**Status Definitions**:
- `idle`: Agent is not actively working on any task
- `processing`: Agent is analyzing tasks or generating plans
- `working`: Agent is actively executing task-related operations
- `error`: Agent encountered an error and is not operational

**Polling Behavior**:
- Refetch interval: 30 seconds (NFR-001)
- Background refetch enabled
- Stale-while-revalidate caching

### Activity Log

A chronological record of system events and actions.

```typescript
// src/types/activity.ts

export type ActivityEventType =
  | 'task_created'
  | 'task_updated'
  | 'plan_generated
  | 'system_started'
  | 'system_error';

export interface ActivityLog {
  id: string;            // UUID v4
  timestamp: string;     // ISO-8601 datetime
  eventType: ActivityEventType;
  message: string;       // 1-500 characters
  metadata?: Record<string, any>;  // Optional structured data
}

export interface ActivityLogQuery {
  limit?: number;        // Default: 50, Max: 100
  offset?: number;       // Default: 0
}
```

**Event Type Definitions**:
- `task_created`: New task was created by user or system
- `task_updated`: Task status or details were modified
- `plan_generated
- `system_started`: AI agent was initialized
- `system_error`: System encountered an error

## Zod Validation Schemas

```typescript
// src/lib/validations.ts

import { z } from 'zod';

export const TaskStatusSchema = z.enum(['pending', 'in-progress', 'completed', 'cancelled']);

export const TaskSchema = z.object({
  id: z.string().uuid(),
  title: z.string().min(1).max(200),
  description: z.string().max(2000).optional(),
  status: TaskStatusSchema,
  createdAt: z.string().datetime(),
  updatedAt: z.string().datetime(),
});

export const CreateTaskInputSchema = z.object({
  title: z.string().min(1).max(200),
  description: z.string().max(2000).optional(),
});

export const PlanSectionSchema = z.object({
  title: z.string().min(1).max(100),
  steps: z.array(z.string().min(1).max(500)).min(1).max(20),
});

export const PlanSchema = z.object({
  id: z.string().uuid(),
  taskId: z.string().uuid(),
  sections: z.array(PlanSectionSchema).min(1).max(10),
  summary: z.string().min(1).max(500),
  createdAt: z.string().datetime(),
});

export const AgentStatusSchema = z.enum(['idle', 'processing', 'working', 'error']);

export const SystemStateSchema = z.object({
  status: AgentStatusSchema,
  lastActivityAt: z.string().datetime(),
  currentTaskId: z.string().uuid().nullable(),
});

export const ActivityEventTypeSchema = z.enum([
  'task_created',
  'task_updated',
  'plan_generated
  'system_started',
  'system_error',
]);

export const ActivityLogSchema = z.object({
  id: z.string().uuid(),
  timestamp: z.string().datetime(),
  eventType: ActivityEventTypeSchema,
  message: z.string().min(1).max(500),
  metadata: z.record(z.any()).optional(),
});
```

## Type Safety Guarantees

1. **No `any` types** in core entity definitions
2. **Strict null checks** enabled in `tsconfig.json`
3. **Zod schemas** for runtime validation of API responses
4. **Discriminated unions** for status types (TaskStatus, AgentStatus)
5. **Branded types** for IDs if needed (future enhancement)
